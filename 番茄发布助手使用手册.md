# 番茄发布助手使用手册

## 1. 工具用途

这个工具用于把本地 `.txt` 小说章节半自动发布到番茄小说作者后台。

它不保存账号密码，也不调用非公开接口。你需要在脚本打开的浏览器里登录番茄作者后台，之后由脚本自动填写章节、保存草稿、发布草稿或直接发布。

## 2. 支持能力

- 本地章节质量检查。
- 自动读取任意小说章节目录。
- 自动识别章节号、章节名和正文。
- 自动填写章节号、章节名、正文。
- 自动删除正文第一行的“第 x 章 标题”。
- 上传草稿。
- 上传并发布。
- 从草稿箱按章节范围发布已保存草稿。
- 草稿箱翻页查找目标章节。
- 自动处理错别字提示、内容检测方式、AI 选项、确认发布弹窗。
- Web 页面实时查看日志。
- 查看草稿上传到哪一章、发布到哪一章。
- 创建定时发布任务。
- 任务完成后自动关闭脚本打开的浏览器。

## 3. 安装和启动

```powershell
npm install
npm run web
```

打开：

```text
http://localhost:3899
```

## 4. 章节目录要求

推荐结构：

```text
某本书/
  chapters/
    001-第一章 标题.txt
    002-第二章 标题.txt
```

也支持每 20 章一个子文件夹：

```text
某本书/
  chapters/
    001-020/
      001-第一章 标题.txt
      002-第二章 标题.txt
```

正文第一行推荐写：

```text
第1章 标题

正文内容……
```

脚本会自动拆成：

- 章节号：`1`
- 章节名：`标题`
- 正文：删除第一行后上传

## 5. Web 控制台字段

`章节目录`：本地 `chapters` 文件夹路径。

`书名`：用于日志和断点记录。可以手动填，也可以从目录推断。

`开始章节 / 结束章节`：本次手动任务处理的章节范围。

`模式`：

- `上传草稿`：只把章节写入后台并保存为草稿。
- `发布草稿箱`：从草稿箱找到已有草稿并发布。
- `本地检查`：只检查本地 txt，不打开浏览器。
- `直接发布`：先填写章节内容，再直接走发布流程。
- `仅当前页发布`：适合当前页面已经填好内容时使用，不会自动上传正文。

`作品后台 URL`：定时任务自动打开的目标页面。

- 发布草稿箱：填章节管理页或草稿箱页。
- 上传并发布：填新建章节页。

`新建章节 URL`：可选。用于上传多章时，每章保存后打开新的新建章节页面，避免覆盖上一章。

## 6. 定时发布

操作步骤：

1. 填写章节目录。
2. 填写书名。
3. 填写作品后台 URL。
4. 选择间隔：`2 / 30 / 60 / 120 / 240` 分钟。
5. 选择定时模式。
6. 设置每次发布章节数。
7. 设置截止章节。
8. 点击“创建定时发布”。

定时模式：

- `发布草稿箱`：从草稿箱找对应章节并发布。适合已经提前上传好草稿的情况。
- `上传并发布`：先填入章节内容，再直接点击发布。适合希望到时间自动上传并发布的情况。

定时任务会读取当前“已发布到第几章”，然后自动计算下一批。例如已发布到 47 章，每次发布 1 章，下一次会处理第 48 章。

如果当前已有任务运行，本轮定时任务会跳过，等待下一轮。

定时任务保存在本地 `.fanqie-schedules.json`，不会上传到 GitHub。

## 7. 浏览器注意事项

脚本会打开本机 Edge 浏览器并使用独立的浏览器配置目录。

任务运行时，尽量不要额外手动打开同一个自动化 Edge 页面，否则可能影响页面定位或配置目录占用。

任务完成后，脚本会自动关闭它打开的浏览器，方便下次定时任务继续运行。

首次使用时，需要在脚本打开的浏览器里手动登录番茄作者后台。

## 8. 推荐发布流程

稳妥流程：

1. 先用本地检查检查章节。
2. 小范围上传 1-3 章草稿。
3. 确认草稿内容没问题。
4. 批量上传草稿。
5. 用“发布草稿箱”或定时“发布草稿箱”发布。

更自动化流程：

1. 先用本地检查检查章节。
2. 在 Web 页面选择“上传并发布”。
3. 设置定时任务。
4. 填新建章节页作为作品后台 URL。
5. 到时间后自动上传并发布下一批。

## 9. 常用命令

本地检查：

```powershell
npm run fanqie -- --chapters "D:\novels\某本书\chapters" --dry-run --start 1 --end 10
```

上传草稿：

```powershell
npm run fanqie -- --chapters "D:\novels\某本书\chapters" --draft --start 1 --end 3 --confirm-each --reset
```

从草稿箱发布：

```powershell
npm run fanqie -- --chapters "D:\novels\某本书\chapters" --publish-drafts --start 1 --end 3 --confirm-each --reset
```

上传并发布：

```powershell
npm run fanqie -- --chapters "D:\novels\某本书\chapters" --upload-and-publish --start 1 --end 3 --confirm-each --reset
```

## 10. 本地检查

本地检查不会打开浏览器，只检查 txt 文件。

检查内容包括：

- 是否缺章。
- 是否重复章节号。
- 标题是否为空。
- 正文是否为空。
- 正文字数是否低于最低字数。
- 是否有明显重复段落。
- 相邻章节相似度是否过高。

## 11. 断点记录

断点记录保存在：

```text
.fanqie-runs
```

Web 页面会根据断点记录显示：

- 草稿上传到：第几章
- 已发布到：第几章

重新跑当前范围：加 `--reset`。

忽略断点但不删除记录：加 `--no-resume`。

## 12. 页面诊断

如果脚本找不到输入框、编辑器或按钮，可以使用诊断模式：

```powershell
npm run fanqie -- --chapters "D:\novels\某本书\chapters" --draft --start 1 --end 1 --inspect-page --reset
```

诊断会导出：

- 页面截图。
- HTML。
- 输入框数量和属性。
- 编辑器数量。
- 当前页面 URL。

诊断文件保存在 `.fanqie-runs`。

## 13. 常见问题

### 定时任务打开 blank 页面

创建定时任务时填写“作品后台 URL”。发布草稿箱填章节管理/草稿箱页，上传并发布填新建章节页。

### 定时任务没有自动继续

新版定时任务会自动跳过“回车继续”。如果是旧任务，删除后重新创建。

### 发布一章后找不到下一章

草稿箱章节过多时脚本会自动翻页查找。如果仍失败，查看日志和失败截图。

### 下一章覆盖了上一章

不要把保存后的草稿编辑 URL 当成新建章节 URL。要使用真正的新建章节页面，或保留章节管理页让脚本点击“新建章节”。

### 草稿箱没有文字“发布”按钮

番茄后台使用的是图标。脚本会点击草稿行右侧的编辑图标进入草稿编辑页，再执行发布流程。

### 任务完成后浏览器还开着

新版脚本处理完成后会自动关闭脚本打开的浏览器。

## 14. 文件说明

`fanqie-publisher.js`：核心自动化脚本。

`web-server.js`：Web 控制台后端。

`public/`：Web 控制台页面。

`.fanqie-runs`：本地断点、日志、失败截图和诊断文件目录，不上传 GitHub。

`.fanqie-schedules.json`：本地定时任务配置，不上传 GitHub。

`fanqie.config.example.json`：配置文件示例。
