# 番茄发布助手 · AI小说工作室

这是一个面向中文网文生产和番茄小说作者后台的本地工作台。它把“AI小说生产、正文质检、批改返修、草稿上传、定时发布”放在同一个 Web 控制台里，适合用本地章节文件管理长篇连载。

工具不保存账号密码，不调用非公开接口。发布相关动作都通过本机浏览器模拟人工操作完成；使用前需要先在浏览器里登录番茄作者后台。

## 核心能力

- AI小说工作室总控台：作品库、生产进度、发布进度、模型配置、健康检查。
- 新书生产：搭建故事系统，生成总设定、章纲和前 10 章。
- 批量续写：按所选章节数继续生成正文。
- 正文检查：选择不同作品后检查指定章节范围。
- 自动修复格式：清理重复章号、开头空行等确定性格式问题。
- 清理章节引用：删除正文里类似“第12章末”“第12章中”的生产痕迹。
- 字数检查：按纯文字口径检查章节是否达标。
- AI剧情批改：对指定章节输出连贯性、人物动机、设定、节奏、钩子问题。
- 修改稿流程：先生成 `_ai_revisions` 修改稿，人工确认后再覆盖原文，覆盖前自动备份。
- 作品文件夹入口：从页面直接打开当前作品目录。
- 发布助手：上传草稿、发布草稿箱、上传并发布、本地 dry-run。
- 断点记录：追踪每本书上传到哪章、发布到哪章。
- 定时发布：按固定间隔自动推进下一批章节。

## 页面入口

启动服务：

```powershell
npm install
npm run web
```

打开 AI 小说工作室：

```text
http://localhost:3899/studio.html
```

打开番茄发布助手：

```text
http://localhost:3899/
```

## 推荐工作流

1. 在 AI 小说工作室配置云端或本地 OpenAI 兼容模型。
2. 新建作品，先搭故事系统，再生成前 10 章。
3. 在“正文检查与批改”里选择作品和章节范围。
4. 执行格式修复、清理章节引用、字数检查。
5. 对最近 5-10 章做 AI 剧情批改。
6. 生成修改稿，人工查看 `_ai_revisions` 后再确认覆盖。
7. 发布前先 dry-run，再小批量上传草稿。
8. 后台人工检查无误后，再批量上传或创建定时发布。

## 章节目录格式

推荐结构：

```text
某本书/
  00-总设定.txt
  01-章纲.txt
  generation_state.json
  chapters/
    001-020/
      001-第1章 标题.txt
      002-第2章 标题.txt
```

正文文件推荐格式：

```text
第1章 标题

正文内容……
```

发布脚本会识别：

- 章节号：来自文件名前缀。
- 章节名：来自第一行标题。
- 正文：上传时会去掉第一行标题。

## 常用命令

本地检查：

```powershell
npm run fanqie -- --chapters "D:\novels\某本书\chapters" --dry-run --start 1 --end 10
```

上传草稿：

```powershell
npm run fanqie -- --chapters "D:\novels\某本书\chapters" --draft --start 1 --end 3 --confirm-each --reset
```

从草稿箱发布：

```powershell
npm run fanqie -- --chapters "D:\novels\某本书\chapters" --publish-drafts --start 1 --end 3 --confirm-each --reset
```

上传并发布：

```powershell
npm run fanqie -- --chapters "D:\novels\某本书\chapters" --upload-and-publish --start 1 --end 3 --confirm-each --reset
```

## 定时发布

在 Web 控制台里创建定时任务：

1. 填写章节目录、书名、作品后台 URL、截止章节。
2. 选择间隔：2/30/60/120/240 分钟。
3. 选择定时模式。
4. 设置每次发布章节数。
5. 点击“创建定时发布”。

定时模式说明：

- `发布草稿箱`：从草稿箱查找对应章节并发布。
- `上传并发布`：先上传本地章节内容，再走发布流程。

## 安全边界

- 不保存番茄账号密码。
- 不调用番茄非公开接口。
- 覆盖正文前会生成备份。
- 打开文件夹功能只允许打开当前工作区内的作品目录。
- AI 修改稿不会自动覆盖原文，必须人工确认。

## 适用场景

- 本地管理长篇网文章节。
- AI 辅助生成设定、章纲和正文。
- 发布前批量检查字数、格式、重复和生产痕迹。
- 小批量草稿上传和定时发布。
- 维护多本小说的生产和发布进度。
